AJAX의 개념과 이용하기

프론트엔드는 우리가 보는 화면을 말하고 백엔드는 이 화면에 보여줄 정보들을 처리하는 곳을 말한다. 프론트엔드에서 1번 글을 조회하면 1번글의 데이터를 달라고 백엔드에 요청한다. 그러면 백엔드는 DB를 조회하여 1번 글에 해당하는 제목, 내용 등의 데이터를 가져오고 프론트엔드에 전달하여 그 데이터를 표시해주는 역할을 하는데, 이때 프론트엔드가 백엔드에 요청을 할 때 어떤 규칙에 맞게 요청을 해야한다. 그러한 요청을 API(Application Programming Interface, 응용 프로그램 프로그래밍 인터페이스)라고 부른다. 여기서 오픈 API라고하면 백엔드를 우리가 직접 만들지 않아도 누군가 이미 만들어 놓고 규칙을 공개한 것을 말한다. 우리는 이 백엔드의 주소와 사용 규칙만 알면 우리가 만든 것이 아니라도 마음대로 사용할 수 있게 되는 것이다.

서버에 정보를 달라고 하는 것을 요청(request)라고 부르고, 찾은 정보를 내 사이트가 받는 것을 응답(response)라고 한다. 이러한 요청과 응답은 API마다 정해진 형식에 맞추어 전달 되는데 이러한 형식이 작성된 문서를 API 가이드 라고 부른다. 전송방식에는 크게 GET과 POST로 나눌 수 있다. GET 방식은 쉽게 말해서 주소창에 모든 정보를 담아 정보를 전달하는 방식이고 POST 방식은 주소창이 아니라 내부적으로 정보를 전송하는 방식이라고 생각하면 된다. 형식은 응답이 오는 형태로 보통 대부분 JSON 형식으로 날아오게 된다. 가끔 XML같은 형식도 쓰이긴 하지만 웬만한 경우에는 데이터를 다루면 보통 JSON을 쓰게된다. 우리가 일반적으로 쓰는 배열이나 객체의 정보로 담겨져 있기 때문이다.

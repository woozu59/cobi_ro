반응형 추가 css 작성 시 주의점


😊뷰포트 삽입
    <meta name="viewport" content="width=device-width, initial-scale=1.0">을 html에 삽입


😊media query 관련 선택 
    - css의 하단에 붙일것인가?
        @media screen and (가로 사이즈 조건){
            css 기술
        }
    - 새 반응형 css 파일을 만들것인가?
        <link rel="stylesheet" media="all and (가로 사이즈 조건)" href="반응형css경로" />
 

😊PC에서 사용한 선택자(css selector)를 그대로 쓴다.
    * 작성하면서 드는 생각 *
    - css네이밍을 신경써야겠군
    - css selector좀 더 익혀야겠군
    - css를 체계적으로 정리하면서 써야겠군

😊적용이 안됨?
    - 변경하고자하는 반응형 css선언들이 모든 css를 통틀어 맨 아래 순서에 있나 확인
    - media query에서 설정한 가로 조건이 맞는가?
    - PC에서 선언한 css selector를 그대로 옮겨왔는가?
    - 오타확인

😊되긴되는데 이상하게 됨?
    - pc css를 먼저 읽었기 때문에 간섭받고 있다는 것을 인지.
    - 변경하고자 하는 선언들을 모두 작성하여 변경값을 적어준다.
    - 반응형에서 사용하지 않을 pc의 속성은 반응형 css에서 각 속성의 default값으로 변경


😊수정금지
    - PC의 css는 이미 잘 돌아가고 있다. 반응형을 작업한다고 PC의 속성값을 변경할 때 주의해야 한다.
    - 반응형에 적어야 할 내용은 반응형에다만 작성하자
      (PC의 속성을 반응형에서 갱신한다)
    - 완전 다시짜야하는경우(미숙해서 잘못 짠 경우)엔 수정하지만, 그게 아니라면 PC는 굳이 건드리지 않는다.


😊멘탈관리
    - 원래 고치는게 많다. 여러분들이 잘못한게 아니라 적게 알았을 뿐이다.
    - 많은 번뇌는 자양분이 된다, 하지만 반복을 얼마까지 할것인가에 대한 기준을 갖기 바람.
    - 너무 많이 같은 실수를 반복하지 말자.
